# PRD 최소 가이드
*빠른 시작을 위한 핵심만 담은 PRD 작성 가이드*

**버전**: 1.0.0 | **타겟**: 경험 많은 개발자

---

## 📋 필수 질문 (3-5개만)

### 1. 목적
```
이 기능의 주요 목적은?
A. 사용자 경험 개선
B. 운영 효율성 향상
C. 매출/전환율 증대
D. 기술 부채 해소
E. 기타: [입력]
```

### 2. 대상 사용자
```
주요 사용자는?
A. 일반 최종 사용자
B. 관리자
C. 개발자/API 사용자
D. 내부 팀원
E. 복합 (여러 유형)
```

### 3. 기능 범위
```
기능의 복잡도는?
A. 단순 (CRUD)
B. 중간 (멀티 스텝)
C. 복잡 (외부 연동/실시간)
```

### 4. 수락 기준
```
완료 기준은?
A. 기본 동작 확인
B. 단위/통합 테스트 통과
C. 성능 벤치마크 충족
D. 사용자 피드백 수집
```

### 5. 범위 제외
```
이번에 하지 않을 것은?
- [항목 1]
- [항목 2]
```

---

## 📄 PRD 구조

```markdown
# [Feature Name] PRD

## 목표
[1-2문장]

## 사용자 스토리
- As [user], I want [action] so that [benefit]

## 기능 요구사항
1. [필수 기능 1]
2. [필수 기능 2]

## 수락 기준
- [ ] [조건 1]
- [ ] [조건 2]

## 범위 제외
- [항목]

## 성공 지표
- [측정 가능한 지표]
```

---

## 💡 팁

**빠른 작성법**:
1. 5개 질문 답변 (2분)
2. PRD 템플릿 채우기 (5분)
3. 개발 시작 (즉시)

**총 소요**: ~10분

---

**저장 위치**: `tasks/prds/XXXX-prd-[feature-name].md`
