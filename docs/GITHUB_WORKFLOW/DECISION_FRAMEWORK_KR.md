# 깃허브 워크플로우 의사결정 프레임워크

**목적**: 로컬 PRD 워크플로우 → GitHub 네이티브 마이그레이션 여부 결정

**예상 시간**: 10-15분

---

## 🎯 3가지 핵심 질문

### Q1. 여러 연결된 레포지토리를 관리하나요?

**예시**:
- SSO 시스템 + 여러 앱 (VTC_Logger, contents-factory)
- 공유 SDK/라이브러리 + 사용하는 프로젝트들
- 마이크로서비스 아키텍처

**GitHub 네이티브의 이점**:
- ✅ 자동 크로스 레포 이슈 링크
- ✅ SDK 업데이트 시 종속 앱에 자동 알림
- ✅ 통합 프로젝트 보드로 전체 시스템 시각화
- ✅ "A 업데이트 → B도 업데이트" 추적 자동화

**답변이 "예"면**: ⭐⭐⭐⭐⭐ (강력 추천)

---

### Q2. 메인 PC 외에서도 코딩하나요?

**예시**:
- 노트북 + 데스크톱 양쪽 사용
- 회사 + 집
- 출퇴근 중 모바일에서 이슈 확인/생성

**GitHub 네이티브의 이점**:
- ✅ 웹 브라우저 어디서나 접근
- ✅ 모바일 앱으로 진행 상황 확인
- ✅ 로컬 파일 동기화 불필요
- ✅ Git 충돌 없음

**답변이 "예"면**: ⭐⭐⭐⭐ (추천)

---

### Q3. 6개월 내 팀 협업 가능성?

**예시**:
- 프리랜서 채용 계획
- 오픈소스로 전환 고려
- 회사 프로젝트로 확장 가능성

**GitHub 네이티브의 이점**:
- ✅ 팀원 추가 시 워크플로우 변경 불필요
- ✅ 내장 코드 리뷰 (PR 댓글)
- ✅ 역할 기반 권한 (GitHub 기본)
- ✅ 전문적인 포트폴리오 (채용 시 유리)

**답변이 "예"면**: ⭐⭐⭐⭐ (추천)

### 시나리오별 ROI

- 투자: 5시간
→ ROI: -40% (손실) ⭐
```

#### 시나리오 C: 단일 레포, 원격 작업 있음
```

## ⚖️ 위험 분석

### 낮은 위험 ✅
- **되돌리기 가능**: GitHub 이슈를 마크다운으로 내보내기 가능
- **무료**: GitHub Free는 무제한 프라이빗 레포 지원
- **락인 없음**: 언제든 로컬 파일로 복귀 가능
- **점진적 전환**: 하이브리드 방식으로 시작 가능

### 고려할 점 ⚠️
- **학습 곡선**: GitHub CLI + 새 워크플로우 (30분-1시간)
- **인터넷 필수**: 오프라인에서는 이슈 생성 불가
- **초기 시간**: 설정에 3시간 필요
- **도구 의존**: GitHub 서비스에 의존

## 🎯 시나리오별 추천

### 1. SSO + 멀티앱 (당신의 경우) ✅✅✅✅✅

**상황**:
- sso-system 레포 + VTC_Logger + contents-factory
- SDK 업데이트 추적 필요
- 각 앱의 SSO 통합 상태 관리

**추천**: **즉시 완전 마이그레이션**

**이유**:
1. 크로스 레포 자동화만으로도 ROI 정당화
2. "SSO 업데이트 → 앱들 업데이트" 자동 추적
3. SDK 버전 불일치 버그 제거
4. 통합 대시보드로 전체 시스템 한눈에

**예상 ROI**: 900% (연 50시간 절감)

---

### 2. 단일 레포, 1인 개발 ⚠️

**상황**:
- 한 개의 레포만
- 원격 작업 없음
- 팀 확장 계획 없음

**추천**: **하이브리드 (GitHub Projects만)**

**이유**:
1. 전체 마이그레이션 오버헤드가 ROI를 초과
2. 시각화만 필요하면 Projects로 충분
3. 로컬 PRD 파일 유지
4. 필요시 점진적 확장

**예상 ROI**: -40% ~ +50%

---

### 3. 팀 프로젝트 ✅✅✅✅✅

**상황**:
- 2명 이상 개발자
- 코드 리뷰 필요
- 작업 분담 필요

**추천**: **즉시 완전 마이그레이션**

**이유**:
1. 로컬 워크플로우는 협업 불가
2. PR 기반 코드 리뷰 필수
3. 작업 가시성 필수
4. 팀 생산성 급상승

**예상 ROI**: 1000%+ (팀원당 월 10시간+ 절감)

### 🔄 점진적 마이그레이션 (2/3 조건 충족 시)

**1주차**: GitHub Projects 추가 (시각화만)
```bash
# Project 생성
gh project create --title "My Project"

# 수동으로 이슈 생성 (로컬 PRD 유지)
gh issue create --title "Feature X"
```

**2주차**: 새 기능은 이슈로 시작
```bash
# 새 기능
gh issue create --template feature-prd.yml

# 기존 PRD 유지
```

**1개월차**: 기존 PRD 마이그레이션
```bash
python scripts/migrate_prds_to_issues.py tasks/prds/*.md
```

**2개월차**: GitHub Actions 추가 (자동화)
