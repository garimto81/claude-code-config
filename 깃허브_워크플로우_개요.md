## 이게 뭔가요?

**로컬 파일 기반 PRD 워크플로우**에서 **깃허브 네이티브 이슈 중심 개발**로 완전히 전환하는 시스템입니다. 특별히 다음을 위해 설계되었습니다:

- **1인 개발자** ("바이브 코더") 멀티 레포 프로젝트 관리
- **SSO 시스템** + 여러 종속 애플리케이션
- **CLAUDE.md Phase 0-6** 워크플로우 사용자
- **팀 준비 완료** 워크플로우가 필요하지만 팀 오버헤드는 싫은 개발자

```
📁 tasks/prds/0001-prd-feature.md     ← PC에서만 접근

🤖 GitHub Actions                     ← 완전 자동화
⏱️  기능당 5분 오버헤드
```

**절감 시간**: 89% (기능당 40분)

## 30초 데모

```bash
# 기존 워크플로우 (45분)
vim tasks/prds/0001-prd-feature.md            # 10분: PRD 작성
python scripts/generate_tasks.py ...          # 5분: 작업 생성
git checkout -b feature/new-feature           # 수동 브랜치
# ... 코딩 ...
npm version minor                             # 5분: 수동 버전 관리
vim CHANGELOG.md                              # 수동 변경사항 기록
git push && PR 수동 생성                       # 10분: PR 생성
# ... 검사 대기 ...
git merge 수동                                # 5분: 병합
# ... 수동 배포 ...                            # 10분: 배포

# 깃허브 네이티브 워크플로우 (5분)
gh issue create --template feature-prd.yml    # 3분: 폼 작성
./scripts/github-issue-dev.sh 123             # 30초: 브랜치 + PR 자동 생성
# ... 코딩 ...
git commit -m "feat: add feature [#123]"      # 커밋
git push                                       # 푸시
gh pr ready                                    # 준비 완료 표시
# → 자동: 테스트 → 버전 → 병합 → 배포 → 이슈 종료 (자동 2-3분)
```

**당신 작업**: 5분
**자동화 작업**: 3분
**총**: 8분 (기존 45분 대비)

## 핵심 기능

### 1인 개발자 최적화

✅ **수동 승인 없음**: 검사 통과 시 자동 병합
✅ **리뷰 대기 없음**: 당신이 리뷰어
✅ **모든 것 자동화**: 테스트, 버전, 배포
✅ **한 줄 명령어로 시작**: `./scripts/github-issue-dev.sh 123`
✅ **모바일 접근**: 어디서나 상태 확인

### 팀 준비 완료 (미래 대비)

✅ **워크플로우 변경 없음**: 팀원 추가 → 동일 프로세스
✅ **내장 코드 리뷰**: PR 댓글 즉시 사용 가능
✅ **역할 관리**: GitHub 권한이 처리
✅ **감사 추적**: 누가 무엇을 했는지 전체 히스토리

### 크로스 레포 조정

✅ **자동 링크**: 레포 간 이슈 참조
✅ **의존성 추적**: 시각적 의존성 그래프
✅ **SDK 업데이트**: 종속 앱 자동 알림
✅ **통합 대시보드**: 모든 레포를 하나의 프로젝트 보드에

git add .github/ISSUE_TEMPLATE/
git commit -m "docs: GitHub 이슈 템플릿 추가"

## 사용 사례

### 사례 1: 1인 개발, 멀티 레포 SSO 프로젝트 ✅

**당신 상황**:
- SSO 시스템 구축 (sso-system 레포)
- 여러 앱 통합: VTC_Logger, contents-factory
- "SSO 업데이트 → 앱들도 업데이트" 추적 필요

**깃허브 네이티브가 좋은 이유**:
- 크로스 레포 이슈 링크: `your-org/sso-system#123`
- SDK 업데이트 시 앱들에 자동 알림
- 레포 전체를 통합한 프로젝트 보드
- 전문적인 포트폴리오 표현

**ROI**: 높음 (크로스 레포 조정만으로도 정당화됨)

### 사례 2: 1인 개발, 단일 레포 ⚠️

**상황**: 한 레포, 원격 작업 없음, 향후 팀 없음

**추천**: 하이브리드 - GitHub Projects 시각화만

**근거**: 전체 마이그레이션 오버헤드가 정당화되지 않을 수 있음

### 사례 3: 팀 프로젝트 ✅✅✅

**상황**: 여러 개발자

**추천**: 즉시 완전한 깃허브 네이티브

**근거**: 로컬 워크플로우는 협업 지원 안 함

## 자주 묻는 질문

### Q: GitHub Actions를 배워야 하나요?
**A**: 아니요, 이슈/프로젝트만 사용해도 됩니다. Actions는 선택적 자동화입니다.

### Q: 전환 중에도 로컬 PRD 파일을 유지할 수 있나요?
**A**: 예, 하이브리드 방식이 지원됩니다. 점진적으로 마이그레이션하세요.

### Q: 마음에 안 들면?
**A**: 이슈를 마크다운으로 내보내고 되돌리세요. 락인 없음.

### Q: 1인 개발에 과한가요?
**A**: 여러 레포를 관리한다면 아닙니다. 크로스 레포 조정만으로도 정당화됩니다.

### Q: 프라이빗 레포는 유료인가요?
**A**: 아니요, GitHub Free에 무제한 프라이빗 레포 포함됩니다.

### Q: ROI를 보려면 얼마나 걸리나요?
**A**: ~15개 기능 후 손익분기 (2-3개월).
