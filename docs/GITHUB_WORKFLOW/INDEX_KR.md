## 📋 구현 체크리스트

### Phase 0: 결정 (30분)
- [ ] [깃허브_워크플로우_개요.md](../깃허브_워크플로우_개요.md) 읽기
- [ ] [깃허브_의사결정_프레임워크.md](깃허브_의사결정_프레임워크.md)로 결정
- [ ] 팀/동료와 논의 (해당 시)

### Phase 1: 기본 설정 (1시간)
- [ ] GitHub CLI 설치 (`gh --version`)
- [ ] 라벨 생성 (`scripts/setup-github-labels.sh`)
- [ ] 이슈 템플릿 커밋 (`.github/ISSUE_TEMPLATE/`)
- [ ] GitHub Project 생성

### Phase 2: 첫 사이클 (1시간)
- [ ] 첫 이슈 생성 (웹 또는 CLI)
- [ ] 자동화 스크립트 테스트 (`github-issue-dev.sh`)
- [ ] 코드 → 커밋 → 푸시
- [ ] PR 병합 완료

### Phase 3: 기존 데이터 마이그레이션 (선택, 2시간)
- [ ] `migrate_prds_to_issues.py` 실행
- [ ] 기존 PRD → 이슈 변환 확인
- [ ] 로컬 파일 아카이브

### Phase 4: 고급 자동화 (선택, 4시간)
- [ ] GitHub Actions 워크플로우 추가
- [ ] 자동 테스트 설정
- [ ] 자동 배포 설정
- [ ] 크로스 레포 동기화

## 🔧 문제 해결 가이드

### 문제: GitHub CLI 인증 실패
**해결**:
```bash
gh auth status
gh auth login --web
```
**참조**: [깃허브_빠른시작.md - 문제해결](../깃허브_빠른시작.md#문제-해결)

### 문제: 이슈 템플릿이 안 보임
**해결**:
1. `.github/ISSUE_TEMPLATE/*.yml` 커밋 확인
2. `git push` 후 5분 대기
3. GitHub 웹에서 Issues → New Issue 확인

**참조**: [깃허브_빠른시작.md](../깃허브_빠른시작.md#3단계-issue-templates-추가-이미-완료)

### 문제: 스크립트 실행 권한 에러 (Windows)
**해결**:
```bash
# Git Bash 사용
bash scripts/github-issue-dev.sh 1

# 또는 수동 실행
gh issue view 1
git checkout -b feature/issue-1
gh pr create --draft
```

### 문제: ROI가 기대보다 낮음
**분석**:
- 단일 레포? → 하이브리드 방식 고려
- 크로스 레포 없음? → 전체 마이그레이션 불필요
- GitHub Actions 미사용? → Phase 2-6 자동화 추가

**참조**: [깃허브_의사결정_프레임워크.md](깃허브_의사결정_프레임워크.md)
