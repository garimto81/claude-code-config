# WSOP+ "The Story Hub" 프로젝트 제안서
**세계 최대 포커 토너먼트의 모든 스토리를 실시간으로 연결하다**

---

## 📋 Executive Summary

### 프로젝트 개요

**WSOP+ "The Story Hub"**는 World Series of Poker 토너먼트의 라이브 스토리텔링 중앙 신경망입니다. 현장의 모든 중요한 순간을 실시간으로 포착하고, 방송팀에게 최적화된 편집 환경을 제공하며, 전 세계 팬들에게 '나만의 주인공'을 따라가는 세컨드 스크린 경험을 선사합니다.

### 핵심 가치 제안

| 대상 | 현재 문제 | Story Hub 솔루션 | 기대 효과 |
|------|-----------|-----------------|-----------|
| **방송 프로덕션** | 스토리 발굴 수동 의존, 클립 검색 15분+ | 자동 스토리 수집, 즉시 영상 접근 | **편집 시간 55% 단축** |
| **현장 스포터** | 산발적 기록, 피드백 부재 | 통합 플랫폼, 실시간 조율 | **누락률 10% → 2%** |
| **전세계 팬** | 수동적 시청만 가능 | My Stable, 실시간 업데이트, 참여 | **앱 체류 시간 5배** |

### 투자 대비 효과

- **개발 기간**: 12주 (Phase 1)
- **핵심 팀**: 5명 (PM 1, 개발 3, 디자인 1)
- **기대 효과**:
  - 방송 콘텐츠 생산성 **2배 향상**
  - WSOP+ 앱 일간 활성 사용자(DAU) **3배 증가**
  - 팬-방송 연결 강화로 **브랜드 가치 상승**

## 🎯 프로젝트 배경 및 필요성

### 현재 도전 과제

#### 1. 방송 프로덕션의 병목

```
현재 워크플로우:
현장 발생 → 제작진 목격/전달 → 클립 수동 검색(15분) → 편집(30분) → 송출
         ↑                   ↑
      운에 의존          시간 낭비

문제점:
• 중요한 핸드 놓침 (추정 20-30%)
• 클립 검색에 전체 시간의 40% 소비
• 실시간성 부족 (핸드 발생 후 45분+)
```

#### 2. 팬 경험의 한계

```
현재 팬 상황:
• 방송 시청만 가능 (수동적)
• 관심 선수 추적 불가
• 토너먼트 전체 맥락 파악 어려움

결과:
• WSOP+ 앱 평균 세션: 3분
• 재방문율: 30%
• 참여도: 거의 없음
```

#### 3. 데이터 손실

```
토너먼트 중 발생하는 스토리의 80%가 기록되지 않음
→ 방송 후 재구성 불가
→ 아카이브 가치 하락
→ 재활용 불가
```

### Story Hub가 해결하는 방법

```
통합 플랫폼 구축:
[8명 스포터] → [중앙 Hub] → [3개 출력]
  실시간 입력      자동 처리     1. 방송 편집팀
                                2. 팬 앱
                                3. 데이터 아카이브

핵심 메커니즘:
✓ 스포터가 현장에서 즉시 로그 입력
✓ WiFi Direct로 영상 자동 전송
✓ 타임스탬프 자동 매칭
✓ BDP는 큐에서 순차 편집
✓ 팬은 방송과 동기화된 업데이트 수신
```

## 💡 솔루션 아키텍처

### 3-Layer 신경망 구조

```
                [The Story Hub]
                 중앙 신경망
        Firebase/Firestore 실시간 DB
                      │
       ┌──────────────┼──────────────┐
       ↓              ↓              ↓
  [Field Core]   [Broadcast]    [Fan Layer]
   스토리 발굴    스토리 설계     스토리 경험
       │              │              │
   스포터 8명      PD/BDP팀       전세계 팬
 (Field Logger) (Admin Dashboard) (Mobile App)
```

### 데이터 플로우

```
T+0: 핸드 발생
  ↓
T+1분: 스포터 입력 (⭐ 평가, 타임마커)
  ↓
T+2분: WiFi Direct 영상 전송 완료
  ↓
T+3분: Hub 자동 매칭, Producer 확인
  ↓
T+4분: BDP 편집 시작
  ↓
T+16분: 편집 완료, YouTube 업로드
  ↓
T+17분: 방송 송출 + 팬 앱 동기화 업데이트
```

**현재 대비 개선**: 45분 → 17분 (**62% 단축**)

## 🔧 핵심 모듈 상세

### Module 1: Field Operations Core

**목적**: 현장에서 스토리 실시간 포착

#### Field Logger App (스포터용 모바일)

**주요 기능**:

1. **KP(Key Player) 우선 표시**
   - Producer 지정 KP 자동 동기화
   - 테이블-좌석 위치 표시
   - Producer Tag (예: #Comeback_Story) 실시간 반영

2. **스토리 가치 평가 시스템**
   ```
   ⭐⭐⭐⭐⭐ - 토너먼트 결정 순간 (버블 버스트, 파이널 진출)
   ⭐⭐⭐⭐   - 방송 하이라이트 (KP 간 대결, 큰 팟)
   ⭐⭐⭐     - Key Hand (기록 가치)
   ⭐⭐       - 참고용
   ⭐         - 데이터만
   ```

3. **타임마커 자동 기록**
   - [HAND START] 버튼 → 시작 시각 기록
   - [HAND END] 버튼 → 종료 시각 기록
   - 핸드 지속 시간 자동 계산

4. **WiFi Direct 영상 전송**
   - 3-앵글 순차 전송 (Wide → Close → Player POV)
   - 백그라운드 자동 업로드
   - 전송 실패 시 SD 백업 알림

5. **선택 정보 입력**
   - 칩스택 (핸드 전/후)
   - 팟 크기
   - 참여 플레이어 수

**스포터 운영**:
- Zone당 1명 배치 (3-4 테이블 담당)
- 인터컴으로 Producer와 실시간 소통
- KP VPIP 25% × 방송 가치 25% = **실제 입력 6.25%** (부담 없음)

---

### Module 2: Broadcast Admin

**목적**: Producer와 BDP의 효율적 스토리 큐레이션 및 편집

#### Producer's Dashboard

**주요 기능**:

1. **KP 관리**
   - 200명 → 5-20명 KP 선정
   - 실시간 태그 추가 (#Heater_Alert, #Comeback_Story 등)
   - 팬 좋아요 Top 10 참고 (강제 아님)

2. **라이브 스토리 모니터링**
   ```
   ┌────────────────────────────────┐
   │ LIVE STORIES (최근 30분)       │
   ├────────────────────────────────┤
   │ 14:30 | Zone 3 | Ivey ⭐⭐⭐⭐⭐│
   │ #Comeback_Story                │
   │ [Add Tag] [Mark Priority]      │
   ├────────────────────────────────┤
   │ 14:28 | Zone 1 | Negreanu ⭐⭐⭐│
   │ [Add Tag]                      │
   └────────────────────────────────┘
   ```

3. **팬 반응 피드백**
   - 실시간 좋아요/만족도 집계
   - "Most Loved Hands" 확인
   - 팬 선호 트렌드 파악

#### BDP Dashboard

**주요 기능**:

1. **편집 큐 (시간순 자동 정렬)**
   ```
   ┌────────────────────────────────┐
   │ PRIORITY QUEUE                 │
   ├────────────────────────────────┤
   │ ⭐⭐⭐⭐⭐ | 14:30 | Ivey       │
   │ Files: 3 | [▶ Wide] [▶ Close] │
   │ [EDIT NOW]                     │
   └────────────────────────────────┘
   ```

2. **원클릭 영상 접근**
   - 구글 드라이브 직접 링크
   - 3-앵글 즉시 프리뷰
   - 타임스탬프 기반 자동 구간 표시

3. **편집 완료 → YouTube 업로드**
   - 편집 완료 버튼 클릭
   - 자동 YouTube 업로드
   - Story Hub에 VOD URL 자동 저장

**처리 프로토콜**:
- 무조건 시간순 편집 (단순, 명확)
- Producer/BDP 동일 공간에서 조율
- 목표 편집 시간: 12-15분/핸드

### Module 3: Fan Interface

**목적**: 팬에게 개인화된 토너먼트 경험 제공

#### My Stable (나만의 선수단)

**기능**:

```
┌─────────────────────────────┐
│ MY STABLE (5/10)            │
├─────────────────────────────┤
│ Phil Ivey                   │
│ 📍 Producer's Pick          │
│ #Comeback_Story             │
│ Stack: $125,000 (↓25%)      │
│ Last: "All-in vs AA"        │
│ 👍 좋아요 1.2K              │
├─────────────────────────────┤
│ Daniel Negreanu             │
│ Stack: $87,500 (↑12%)       │
│ 👍 좋아요 892               │
└─────────────────────────────┘
```

**실시간 업데이트**:
- 칩스택 변화 (방송 시간 기준)
- 중요 핸드 알림
- Elimination 즉시 푸시

#### Key Hand Feed (핵심 핸드 피드)

**기능**:

```
┌─────────────────────────────┐
│ KEY HAND FEED               │
├─────────────────────────────┤
│ Phil Ivey vs Negreanu       │
│ All-in | Pot: $125,000      │
│ #Comeback_Story             │
│ [🎬 Watch VOD]              │
│ 👍 좋아요 245               │
│ [🔥 Amazing] [👍 Good] [👎] │
├─────────────────────────────┤
│ 팬 반응: 🔥 78% | 👍 18%    │
└─────────────────────────────┘
```

**VOD 시청**:
- 방송 송출 후 즉시 제공
- YouTube 무료 스트리밍
- 앱 내 인라인 재생

#### 팬 상호작용 시스템

**1. 좋아요 (KP 후보 추천)**
- 핸드/선수에 좋아요
- Producer Dashboard에 Top 10 표시
- "팬들이 주목하는 선수" 인사이트

**2. 만족도 평가**
- VOD 시청 후 🔥(Amazing) / 👍(Good) / 👎(Meh)
- 실시간 집계
- Producer에게 피드백 → 스토리 기준 개선

**효과**:
- 수동적 시청 → 능동적 참여
- 팬 의견 → 방송 품질 향상
- 커뮤니티 형성

## 📊 기술 스택

### 프론트엔드

| 플랫폼 | 기술 스택 | 이유 |
|--------|-----------|------|
| Field Logger | React Native | 크로스 플랫폼 (iOS/Android), WiFi Direct 지원 |
| Admin Dashboard | React + TypeScript | 실시간 업데이트, 컴포넌트 재사용 |
| Fan App | React Native | 기존 WSOP+ 앱 통합 가능 |

### 백엔드

| 서비스 | 기술 | 이유 |
|--------|------|------|
| 실시간 DB | Firebase Firestore | 실시간 동기화, 확장성, 빠른 구현 |
| 인증 | Firebase Auth | 통합 솔루션 |
| Functions | Cloud Functions | 서버리스, 자동 스케일링 |
| 저장소 | Google Drive API | 대용량 영상 저장 |
| VOD | YouTube Data API | 무료 배포, 글로벌 CDN |

### 인프라

```
WiFi Direct (영상 전송)
    ↓
Google Drive (원본 저장)
    ↓
Firebase (메타데이터 + 실시간 동기화)
    ↓
YouTube (VOD 배포)
    ↓
WSOP+ Fan App (소비)
```

### 보안 및 권한

- **스포터**: Field Logger 앱만 접근
- **Producer**: Admin Dashboard 전체 권한
- **BDP**: 편집 큐 + 업로드 권한
- **팬**: 읽기 전용, 좋아요/평가만
