# Issue 등록 및 솔루션 추천 (Multi-Agent 병렬 버전)

멀티 에이전트가 병렬로 이슈를 분석하고 최적의 해결책을 제안합니다.

## 병렬 워크플로우

```
┌─────────────────────────────────────────┐
│           Supervisor Agent              │
│         (이슈 분석 및 분할)              │
└────────────────┬────────────────────────┘
                 │
    ┌────────────┼────────────┬────────────┐
    ▼            ▼            ▼            ▼
┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐
│ GitHub │  │  Web   │  │Library │  │ Custom │
│ Search │  │ Search │  │ Search │  │  Impl  │
│ Agent  │  │ Agent  │  │ Agent  │  │ Agent  │
└────┬───┘  └────┬───┘  └────┬───┘  └────┬───┘
     │           │           │           │
     └───────────┴───────────┴───────────┘
                 │
                 ▼
         ┌───────────────┐
         │  Aggregator   │
         │ (솔루션 비교)  │
         └───────────────┘
```

## 입력 정보 수집

다음 정보를 사용자에게 질문하세요:

1. **이슈 제목**: 간단한 제목
2. **이슈 설명**: 문제 상황 상세 설명
3. **기술 스택**: 관련 기술/프레임워크 (예: React, Python, Node.js)
4. **제약 조건**: 시간, 비용, 기술 제약 (선택)

## 병렬 분석 프로세스

### Step 1: 문제 분석 (Supervisor)
- 이슈의 핵심 원인 파악
- 영향 범위 분석
- 해결 우선순위 판단
- 4개 서브에이전트에 태스크 할당

### Step 2: 병렬 솔루션 리서치 (4 Agents 동시 실행)

| Agent | 담당 영역 | 검색 대상 |
|-------|----------|-----------|
| **GitHub Agent** | 오픈소스 솔루션 | 유사 이슈, 관련 프로젝트, 검증된 패턴 |
| **Web Agent** | 웹 리소스 | Stack Overflow, 공식 문서, 기술 블로그 |
| **Library Agent** | 라이브러리/패키지 | npm, PyPI, 프레임워크 생태계 |
| **Custom Impl Agent** | 직접 구현 방안 | 아키텍처, 코드 패턴, 구현 비용 분석 |

### Step 3: 솔루션 추천 (최소 3개)

각 솔루션에 대해 다음 형식으로 제시:

```
## 솔루션 [번호]: [솔루션명]

**유형**: 라이브러리 | 서비스 | 직접 구현 | 설정 변경
**난이도**: 상 | 중 | 하
**예상 소요 시간**:
**비용**: 무료 | 유료 (예상 비용)

### 설명
[솔루션 상세 설명]

### 장점
-

### 단점
-

### 참고 링크
- [출처명](URL)
```

### Step 4: 최종 추천

솔루션 비교표 제공:

| 솔루션 | 유형 | 난이도 | 시간 | 비용 | 추천도 |
|--------|------|--------|------|------|--------|
| ... | ... | ... | ... | ... | ⭐⭐⭐ |

**최종 추천**: 상황에 맞는 최적 솔루션 1개 선정 및 이유 설명

## 직접 구현 판단 기준

다음 경우 "직접 구현"을 추천에 포함:
- 기존 솔루션이 요구사항에 맞지 않음
- 보안/규정 상 외부 라이브러리 사용 불가
- 단순한 기능이라 라이브러리 도입이 과도함
- 학습 목적
- 장기적 유지보수 고려 시 자체 구현이 유리

## 이슈 등록 (선택)

사용자가 원할 경우 GitHub Issue 생성:

```bash
gh issue create --title "[이슈 제목]" --body "[이슈 본문]" --label "bug,needs-triage"
```
