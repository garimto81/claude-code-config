# WSOP+ "The Story Hub" 설계 의뢰서
**프로덕션 현장의 혁신을 위한 통합 플랫폼 구축 요청**

---

## 📌 의뢰 배경

### 의뢰자
**WSOP+ Production Team**
- 세계 최대 포커 토너먼트 라이브 방송 제작팀
- 8명 현장 스포터 + 프로듀서 + BDP(방송편집팀) 운영
- 연간 50+ 일 라이브 프로덕션 진행

### 의뢰 목적
현장에서 발생하는 **모든 중요한 스토리를 놓치지 않고**, **방송팀이 빠르게 편집**할 수 있으며, **팬들이 실시간으로 참여**할 수 있는 통합 시스템이 필요합니다.

**현재 우리가 직면한 문제를 해결해주세요.**

---

## 🔴 우리의 현실적인 문제점

### Problem 1: 스토리가 사라진다

**현장 상황**:
```
토너먼트 진행 중 40개 테이블에서 동시다발적으로 핸드 발생
→ 스포터 8명이 각자 Zone(3-4 테이블) 관찰
→ 중요한 순간 포착 시 무전으로 제작진에게 전달
→ 제작진이 카메라 SD 카드에서 해당 영상 찾기 시작
→ 찾는데 15-20분 소요
→ 그 사이 또 다른 중요 핸드 발생
→ 첫 번째 핸드를 찾지 못하고 포기
```

**결과**:
- 추정 **20-30%의 방송 가치 있는 핸드를 놓침**
- 스포터가 "보고만" 하고 기록이 남지 않음
- 나중에 "그때 뭐가 있었는데..." 재구성 불가능

**우리가 원하는 것**:
> **"스포터가 현장에서 버튼만 누르면, 그 순간의 모든 정보와 영상이 자동으로 기록되고, 우리(제작진)가 언제든 즉시 찾아볼 수 있게 해주세요."**

---

### Problem 2: 편집팀이 영상 찾는데 시간을 다 쓴다

**BDP(방송편집팀)의 하루**:
```
09:00 - 토너먼트 시작, 스포터들 무전 대기
10:30 - "Table 12 Phil Ivey 큰 핸드!" (무전)
10:31 - BDP: "몇 시 몇 분?" / 스포터: "지금요!"
10:32 - BDP: 구글 드라이브 → Table 12 폴더 열기
10:33 - 100개 파일 중 시간 확인하며 클릭
10:38 - 3번째 파일에서 발견 (5분 소요)
10:40 - 3개 앵글 찾기 시작 (각기 다른 카메라)
10:47 - 전체 영상 확보 완료 (15분 경과)
10:48 - 편집 시작
11:00 - 편집 완료 (12분)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
총 27분 → 이 중 편집은 12분, 찾는 시간이 15분
```

**우리의 좌절**:
- BDP의 60%의 시간이 "찾기"에 소비됨
- 편집 실력은 좋은데 파일 탐색에 지쳐버림
- 급한 핸드는 "일단 대충" 찾아서 편집 → 품질 하락

**우리가 원하는 것**:
> **"스포터가 기록한 핸드를 BDP가 클릭 한 번에 영상을 보고, 바로 편집을 시작할 수 있게 해주세요. 검색은 시스템이 하게 하고, 우리는 창작에만 집중하고 싶습니다."**

---

### Problem 3: 프로듀서와 스포터 간 조율이 어렵다

**현재 커뮤니케이션**:
```
프로듀서: "Zone 3, Phil Ivey 모든 핸드 다 기록해줘!" (인터컴)
스포터: "네!"
→ 30분 후...
프로듀서: "Ivey 핸드 몇 개나 올라왔어?"
BDP: "하나도 안 왔는데요?"
프로듀서: "Zone 3! 뭐해??"
스포터: "아 죄송합니다, Ivey가 지금 폴드만 하고 있어요..."
프로듀서: "그럼 그렇다고 알려주지!"
```

**문제점**:
- 현장은 시끄럽고 바쁨 (인터컴만으론 한계)
- 프로듀서가 8명 스포터 상황 파악 불가
- 스포터도 다른 Zone 상황 모름
- "기록했다고 했는데 안 올라옴" 빈번

**우리가 원하는 것**:
> **"프로듀서가 화면 하나로 전체 상황을 보고, 중요한 태그를 달면 스포터들이 바로 알 수 있게 해주세요. 인터컴은 보조 수단이고, 시스템이 주 소통 채널이 되어야 합니다."**

---

### Problem 4: 팬들은 우리가 뭘 하는지 모른다

**팬의 경험 (현재)**:
```
팬: WSOP+ 앱 설치 → 방송 시청
    "Phil Ivey 어떻게 되고 있지?" → 모름
    "지금 중요한 순간인가?" → 모름
    "내가 좋아하는 선수 상황은?" → 모름
    → 앱 종료 (3분 만에)
```

**우리의 아쉬움**:
- 우리는 40개 테이블, 200명 선수 데이터를 다 보고 있음
- 이 엄청난 정보를 팬들은 전혀 못 봄
- 방송은 시간당 8-10개 핸드만 보여줌 (전체의 5%)
- 나머지 95%는 그냥 증발

**우리가 원하는 것**:
> **"우리가 현장에서 포착한 스토리를 팬들도 실시간으로 볼 수 있게 해주세요. 방송은 하이라이트, 앱은 전체 스토리. 팬들이 자기가 좋아하는 선수를 골라서 따라갈 수 있다면 앱 사용 시간이 10배는 늘 겁니다."**

   "Ivey - #Comeback_Story 주목"

```

**10시 35분 - 피드백 확인**:
```

### BDP 관점

**10시 35분 - 편집 큐 확인**:
```
내 화면:
┌────────────────────────────────┐
│ EDITING QUEUE (시간순)         │
├────────────────────────────────┤
│ 🔴 10:30 | Ivey All-in        │
│ ⭐⭐⭐⭐⭐ | #Bubble_Burst      │
│ [▶ Wide] [▶ Close] [▶ POV]    │ ← 클릭하면 바로 재생
│ [EDIT NOW]                     │
├────────────────────────────────┤
│ 10:28 | Negreanu ⭐⭐⭐        │
│ Files: 2/3 (POV 아직 전송중)   │
└────────────────────────────────┘
```

**편집 시작**:
1. [EDIT NOW] 클릭
2. 구글 드라이브 영상 즉시 열림 (검색 0초)
3. Wide 앵글 확인 → 좋음
4. Premiere/Final Cut으로 편집 (12분)
5. 완료 후 [UPLOAD TO YOUTUBE] 클릭
6. 자동 업로드 → 시스템에 VOD URL 자동 저장

**총 시간**: 12분 (편집만)
**기존**: 27분 (검색 15분 + 편집 12분)
**절약**: 15분 (55% 단축)

## 🎯 우리가 요청하는 핵심 기능

### 요청 1: 스포터용 Field Logger App (모바일)

**필수 기능**:

| 기능 | 우리의 니즈 | 비고 |
|------|------------|------|
| **KP 자동 표시** | 프로듀서가 지정한 주요 선수 자동 싱크 | 매일 아침 업데이트 |
| **원터치 기록** | [HAND START] → [HAND END] 두 번만 누르면 완료 | 30초 이내 입력 |
| **⭐ 평가 시스템** | 5단계 별점으로 스토리 가치 판단 | 직관적 UI |
| **WiFi Direct 자동 전송** | 영상을 백그라운드로 자동 업로드 | 스포터는 신경 안 써도 됨 |
| **프로듀서 태그 수신** | 실시간 지시사항 확인 | 인터컴 보조 |
| **칩스택 입력** | 선택사항 (여유 있을 때만) | 팬 경험 향상용 |

**운영 환경**:
- 8명 스포터 동시 사용
- 토너먼트장 WiFi (불안정할 수 있음 → 백업 필요)
- iOS/Android 둘 다 지원

**우리가 체크할 사항**:
- ✅ 햇빛 아래서도 화면 잘 보이나요?
- ✅ 배터리 8시간 이상 가나요?
- ✅ WiFi 끊겨도 데이터 안 날아가나요?

## 📐 기술적 요구사항 (필수)

### 데이터 흐름

```
[스포터 입력]
    ↓ (즉시)
[중앙 데이터베이스] ← 실시간 동기화
    ↓
[프로듀서 확인 + 태그]
    ↓
[BDP 편집]
    ↓
[YouTube 업로드]
    ↓
[팬 앱 업데이트] ← 방송 시간 동기화
```

### 영상 처리

```
[카메라 (40대)]
    ↓
[스포터 스마트폰 WiFi Direct]
    ↓
[구글 드라이브 자동 업로드]
    ↓ (타임스탬프 자동 매칭)
[메타데이터 연결]
    ↓
[BDP 즉시 접근]
    ↓
[편집본 YouTube 업로드]
```

**중요**:
- WiFi 실패 시 SD 카드 백업 경로 필요
- 타임스탬프 매칭 실패 시 수동 싱크 가능해야 함
- 영상 용량: 4K, 파일당 2-5GB (3-앵글 합산 최대 15GB)

### 시간 동기화

**현실적 제약**:
- 카메라 40대 시간 동기화 필요 (촬영 전 수동 설정)
- 스포터 스마트폰 8대 시간 동기화
- 중앙 서버 시간

**요구사항**:
- 촬영 시작 전 "시간 동기화 체크리스트" 제공
- 오차 ±5초 이내 허용
- 매칭 실패 시 BDP가 수동으로 확인 가능한 인터페이스

**토너먼트장**:
- 면적: 축구장 2개 크기
- 테이블: 40개 (Day 1) → 9개 (파이널 테이블)
- 소음: 매우 큼
- WiFi: 불안정 (수천 명 동시 접속)
- 조명: 밝음 (스마트폰 화면 가독성 중요)

**프로덕션 부스**:
- 프로듀서 + BDP 2-3명 동일 공간
- 큰 모니터 3-4개
- 인터컴으로 스포터와 소통
- 긴급 상황 시 육성 조율 가능

### 시간 제약

**우리의 목표**:
- 핸드 발생 → 방송 송출: **17분 이내**
  - 스포터 입력: 1분
  - 영상 전송: 2분
  - 프로듀서 확인: 1분
  - BDP 편집: 12분
  - 업로드: 1분

**현실**:
- 가끔 15분 안에 해야 할 때도 있음 (긴급 상황)
- 대부분은 20분 이내면 OK
- 중요한 건 "검색 시간 제로"

### 사용자 숙련도

**스포터 (8명)**:
- 나이: 25-45세
- 기술 숙련도: 중간 (스마트폰 일반 사용 가능)
- 포커 지식: 높음
- 교육 시간: 사전 2시간, 현장 30분

**프로듀서 (1명)**:
- 나이: 35-50세
- 기술 숙련도: 높음
- 포커 지식: 매우 높음
- 결정 속도: 매우 빠름 (필요)

**BDP (2-3명)**:
- 나이: 25-40세
- 기술 숙련도: 전문가 (영상 편집 전문)
- 포커 지식: 중간
- 편집 도구: Premiere Pro / Final Cut

## 🚧 우리가 예상하는 어려움과 요청사항

### 어려움 1: WiFi 불안정

**현실**:
- 토너먼트장에 수천 명 WiFi 사용
- 가끔 끊김
- 5G도 사람 많으면 느림

**요청**:
- WiFi 실패 시 자동으로 SD 카드 백업 모드 전환
- 스포터에게 "WiFi 실패, SD 카드로 전송하세요" 알림
- 나중에 WiFi 복구되면 자동 재전송

---

### 어려움 2: 시간 동기화

**현실**:
- 카메라 40대를 매일 아침 시간 맞추기 힘듦
- 사람이 하는 일이라 오차 있음

**요청**:
- 촬영 시작 전 "시간 동기화 가이드" 제공
- 오차 ±5초는 허용
- 매칭 실패 시 BDP가 수동으로 "이 영상 맞음" 선택 가능한 UI

---

### 어려움 3: 스포터 숙련도

**현실**:
- 스포터마다 기술 숙련도 다름
- 긴장하면 실수함
- 입력 빼먹을 수 있음

**요청**:
- 초간단 UI (버튼 큼지막하게)
- 입력 빼먹으면 경고 ("핸드 종료 안 했어요")
- 프로듀서가 "스포터 A는 오늘 3건 입력" 한눈에 확인

---

### 어려움 4: 긴급 상황 대응

**현실**:
- 파이널 테이블 버블 버스트 = 10분 안에 방송 내보내야 함
- 시스템 다운되면 큰일

**요청**:
- 시스템 장애 시 최소 기능이라도 작동 (오프라인 모드)
- 긴급 모드 버튼 (프로듀서가 누르면 모든 스포터에게 알림)
- 데이터 백업 (구글 드라이브 장애 대비)

## 📅 일정 및 단계별 요청

### Phase 1: MVP (12주 목표)

**우리가 꼭 필요한 것만**:
- ✅ Field Logger (KP 표시, 타임마커, ⭐ 평가, WiFi 전송)
- ✅ Admin Dashboard (KP 관리, 스토리 피드, 편집 큐)
- ✅ Fan Interface (My Stable, Key Hand Feed, 푸시 알림)
- ✅ VOD 업로드 자동화

**빠져도 되는 것**:
- ⏸️ AI 자동 예측
- ⏸️ 팬 예측 게임
- ⏸️ 고급 분석 대시보드

**파일럿 계획**:
- Week 12에 소규모 토너먼트 1회 테스트
- 스포터 8명 + 프로듀서 + BDP 전원 참여
- 실제 상황에서 검증

---

### Phase 2: 진화 (Phase 1 성공 후)

**우리가 더 원하는 것**:

**자동화**:
- AI가 "이 핸드 중요할 것 같아요" 추천
- 스포터 보조 도구 (놓칠 뻔한 핸드 알림)
- 자동 클립 편집 (BDP는 확인만)

**팬 경험**:
- 스토리 타임라인 (하루 전체 흐름 시각화)
- 예측 게임 (다음 핸드 결과 맞추기)
- 리더보드 (예측 잘하는 팬 순위)

**분석**:
- BDP 효율성 대시보드 (병목 지점 자동 발견)
- 팬 선호도 분석 (어떤 스토리를 좋아하나)
- 스포터 퍼포먼스 (누가 제일 정확하게 평가하나)

**하지만**:
- Phase 1 없이 Phase 2는 의미 없음
- 우선은 기본에 집중

## 🏁 마무리

### 우리의 진심

우리는 **세계 최고의 포커 토너먼트**를 만들고 있습니다.

하지만 기술이 따라오지 않아서 **놓치는 스토리가 너무 많습니다**.

스포터들은 현장에서 뛰어다니며 중요한 순간을 포착하는데, 그 정보가 **증발**합니다.

BDP는 편집 실력은 최고인데, 파일 찾는데 **시간을 다 씁니다**.

팬들은 WSOP를 사랑하는데, 방송 외에는 **아무것도 못 봅니다**.

**이 문제를 해결하고 싶습니다.**

그래서 여러분께 이 시스템 구축을 의뢰합니다.

우리는 프로덕션 전문가입니다. 스토리를 찾고, 편집하고, 전달하는 일은 잘합니다.

하지만 시스템을 만드는 건 **여러분의 영역**입니다.

함께 만들어주세요.

**2024 WSOP Main Event**부터 이 시스템으로 방송하고 싶습니다.

그러면 전 세계 포커 팬들이 **완전히 새로운 경험**을 하게 될 겁니다.
