# 병렬 리서치 (Multi-Agent)

멀티 에이전트를 병렬로 실행하여 복합 리서치를 수행합니다.

## 워크플로우

```
┌─────────────────────────────────────────┐
│           Supervisor Agent              │
│        (주제 분석 및 분할)               │
└────────────────┬────────────────────────┘
                 │
    ┌────────────┼────────────┬────────────┐
    ▼            ▼            ▼            ▼
┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐
│Research│  │Research│  │Research│  │Research│
│Agent 1 │  │Agent 2 │  │Agent 3 │  │Agent 4 │
└────┬───┘  └────┬───┘  └────┬───┘  └────┬───┘
     │           │           │           │
     └───────────┴───────────┴───────────┘
                 │
                 ▼
         ┌───────────────┐
         │  Aggregator   │
         │ (결과 종합)    │
         └───────────────┘
```

## 실행 방법

이 커맨드가 실행되면:

1. **입력 수집**: 리서치 주제와 범위 확인
2. **태스크 분해**: Supervisor가 독립적인 서브태스크로 분할
3. **병렬 실행**: 3-5개 서브에이전트가 동시에 리서치
4. **결과 집계**: Aggregator가 결과 종합 및 중복 제거
5. **최종 보고서**: 구조화된 리서치 보고서 생성

## 사용 시나리오

### 시나리오 1: 기술 비교
```
"React vs Vue vs Angular 비교 분석"

→ Agent 1: React 분석
→ Agent 2: Vue 분석
→ Agent 3: Angular 분석
→ Agent 4: 성능/에코시스템 비교
→ Aggregator: 종합 비교표 + 추천
```

### 시나리오 2: 시장 조사
```
"AI 코딩 어시스턴트 시장 현황"

→ Agent 1: 주요 제품 분석 (GitHub Copilot, Cursor, etc.)
→ Agent 2: 가격/기능 비교
→ Agent 3: 사용자 리뷰/평가
→ Agent 4: 트렌드/미래 전망
→ Aggregator: 시장 보고서
```

## 출력 형식

```markdown
# 리서치 보고서: [주제]

## Executive Summary
[핵심 요약 3-5줄]

## 상세 분석

### 1. [서브주제 1]
[Agent 1 결과]

### 2. [서브주제 2]
[Agent 2 결과]

...

## 비교 분석
| 항목 | 옵션A | 옵션B | 옵션C |
|------|-------|-------|-------|
| ... | ... | ... | ... |

## 결론 및 권장사항
[종합 분석 및 추천]

## 참고 자료
- [출처 1](URL)
- [출처 2](URL)
```
