# PRD 표준 가이드
*대부분의 프로젝트에 적합한 균형잡힌 PRD 작성 가이드*

**버전**: 1.0.0 | **타겟**: 중급 개발자 | **소요**: ~20-30분

---

## 🎯 프로세스

```
1. 초기 요청 수신
   ↓
2. 명확화 질문 (A/B/C/D 옵션)
   ↓
3. PRD 작성
   ↓
4. 사용자 승인
   ↓
5. Phase 1 (개발)
```

---

## 🤔 명확화 질문 (최대 8개)

### 1. 문제/목표
```
Q1. 이 기능의 핵심 목적은?
A. 사용자 경험 개선
B. 운영 효율성 향상
C. 매출/전환율 증대
D. 기술 부채 해소
E. 기타: [입력]

Q2. 성공을 어떻게 측정하나요?
A. 사용자 참여도 증가 (%)
B. 처리 시간 감소 (%)
C. 오류율 감소 (%)
D. 정성적 피드백
```

### 2. 대상 사용자
```
Q3. 주요 사용자 유형은?
A. 일반 최종 사용자
B. 관리자
C. 개발자/API 사용자
D. 내부 팀원
E. 복합 (여러 유형)

Q4. 사용자 기술 수준은?
A. 초급 (기술 지식 거의 없음)
B. 중급 (기본 개념 이해)
C. 고급 (전문가 수준)
D. 혼합
```

### 3. 핵심 기능
```
Q5. 기능의 복잡도는?
A. 단순 (CRUD, 단일 화면)
B. 중간 (멀티 스텝 워크플로우)
C. 복잡 (외부 연동/실시간/고급 로직)

Q6. 가장 중요한 사용 시나리오 3가지는?
1. [시나리오 1]
2. [시나리오 2]
3. [시나리오 3]
```

### 4. 스토리 & 수락 기준
```
Q7. 필수 사용자 스토리를 3-5개로 요약하면?
(형식: As [user], I want [action] so that [benefit])

예시:
- As a blog reader, I want to save articles so that I can read them offline
```

### 5. 범위 & 데이터
```
Q8. 이번에 명확히 제외할 것은?
- [제외 항목 1]
- [제외 항목 2]

Q9. 다루는 주요 데이터 유형은?
A. 사용자 생성 콘텐츠
B. 시스템 설정/구성
C. 트랜잭션 데이터
D. 분석/로그 데이터
E. 외부 API 데이터
```

### 6. 디자인 & 기술
```
Q10. UI/UX 복잡도는?
A. 기존 패턴 재사용
B. 일부 커스텀 필요
C. 완전히 새로운 디자인

Q11. 기술적 제약사항이 있나요?
- [제약사항 1]
- [제약사항 2]
```

## 💡 작성 팁

### 효율적인 작성법
1. **명확화 질문 답변** (5-10분)
   - A/B/C/D 옵션 선택
   - 추가 설명은 필요시만

2. **템플릿 채우기** (15-20분)
   - 필수 섹션 우선
   - 선택 섹션은 필요시만

3. **검토 & 승인** (5분)
   - 사용자 확인
   - 개발 시작

### 체크리스트
- [ ] 문제가 명확히 정의되었나?
- [ ] 사용자 스토리가 구체적인가?
- [ ] 수락 기준이 측정 가능한가?
- [ ] 범위 제외가 명시되었나?
- [ ] 기술 제약사항을 고려했나?
